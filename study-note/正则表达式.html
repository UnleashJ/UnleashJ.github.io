正则表达式与模式匹配

2022/4/16 正则表达式RegExp

最近在学习Vue源码过程中，模板编译章节里碰到一些正则表达式及RegExp类的方法。包括开始标签、属性、结束标签、文本、注释等字符串的正在表达式。重新复习一下正则表达式的相关知识，做一下总结。

# 定义正则表达式

在JavaScript中定义正则表达式有两种方式：

  1. RegExp()构造函数来创建，接收一个字符串参数
  2. 字面量语法，包含在一对/之间的字符 /s$/

在第二个斜杠后面可以设置标志字符，用于控制匹配的方式， /s$/i。或者将标志字符作为RegExp()构造函数的第二个参数，也可以控制匹配方式。

# 字面量字符

所有字母字符和数字字符在正则表达式中都匹配自身的字面值。正则表达式语法通过反斜\作为转义字符。如果想在正在表达式中使用以下英文标点符号的字符，也需要使用反斜杠（\）进行转义（通用规则），因为单独使用他们有页数的含义，各个字符的单独使用时候的具体含义在后面中进行介绍。可以给所有标点字符前面都加上反斜杠（\）。

^ $ . * + + ? = ! : | \ / ( ) [ ] { }

如果使用RegExp构造函数，正则表达式中的任何反斜杠都要写两次，因为字符串也是用反斜杠作为转义字符（RegExp构造函数接收一个字符串参数）。

# 字符类

把一些字面量值放在方括号中可以组成字符类，匹配方括号中的任意字符。

/[abc]/ 匹配a、b、c中的任意一个字符
/[^abc]/ 匹配非a、b、c中的任意一个字符

一些常用的字符类可以使用特殊字符和转义序列进行表示：

元字符 描述
. (opens new window) 查找单个字符，除了换行和行结束符。
\w (opens new window) 查找数字、字母及下划线。等价于[a-zA-Z0-9_]
\W (opens new window) 查找非单词字符。等价于[^a-zA-Z0-9_]
\d (opens new window) 查找数字。等价于[0-9]
\D (opens new window) 查找非数字字符。等价于[^0-9]
\s (opens new window) 查找空白字符。
\S (opens new window) 查找非空白字符。
\p{…} Unicode字符类

# 重复

重复说明符都应用给他们前面的一个字符或者字符类。如果想匹配更复杂的表达式，可以使用圆括号定义一个匹配组，后面介绍。

字符 含义
{n,m} 匹配前项n到m次
{n,} 匹配前项n次或更多次
{n} 匹配前项n次
? 匹配前项零次或一次，即有或没有，{0,1}
+ 匹配前项一次或多次，{1,}
* 匹配前项任意次，{0,}

这些重复字符会尽可能多地匹配（贪婪重复）。在重复字符后面加一个?，可以指定非贪婪重复。

let reg = /a+/
let reg2 = /a+?/
"aaa".match(reg)//['aaa', index: 0, input: 'aaa', groups: undefined]
"aaa".match(reg2)//['a', index: 0, input: 'aaa', groups: undefined]

正则表达式模式的匹配会从字符串的第一个位置开始查找匹配项，下面的正则表达式因为在一开始就找到了匹配项，所以从后续字母开始更短的匹配项（ab）就不在考虑之列了。

let reg3 = /a+?b/
"aaab".match(reg3)//['aaab', index: 0, input: 'aaab', groups: undefined]

# 任选、分组、引用

竖线字符|用于分隔任选模式。/ab|cd|ef/匹配“ab”、“cd”或者"ef"。如果竖线左边的任选模式匹配，则忽略右边的模式，即使右边的模式可以更好地匹配。


圆括号可以把独立的模式分组为子表达式，从而可以被重复说明符当做一个整体，如/java(script)?/。

圆括号的另一个作用是在完整的模式中定义子模式。当正则表达式成功匹配一个目标字符串之后，可以从目标字符串中提取出与圆括号包含的子模式对应的部分。注意下面“+”位置不用，匹配的子模式文本也不同，一个是3，一个是123。

let reg = /[a-z]+(\d)+/
let reg2 = /[a-z]+(\d+)/
"ab123".match(reg)//['ab123', '3', index: 0, input: 'ab123', groups: undefined]
"ab123".match(reg2)//['ab123', '123', index: 0, input: 'ab123', groups: undefined]

回引子表达式，可以使用反斜杠加数字，如\2表示回引第几个子表达式（位置按照左括号计算）。该引用方式引用的是模式匹配的文本，而不是表达式的模式。如/(['"])[^'"]*\1/中的\1引用的是前面第一个括号的具体文本：引号或者双引号；而不是模式['"]。注意在字符类中使用引用是不合法的，/(['"])[^\1]*\1/是错误的。如果不圆括号分组的子表达式生成数字引用，可以使用(?:…)


命名捕获组，可以给每一个圆括号分组起一个名字，引用的时候使用该名字进行引用，可以更好的理解表达式的用途。使用(?<name>…)，分组名字放在<>中间，使用\k<name>可以引用指定名字的模式。上面例子的可以重写为/(?<quote>['"])[^'"]*\k<quote>/。

# 指定匹配位置

有一些正则表达式符号匹配字符间的位置而非实际字符，如常见的^、$，分别锚定字符串开头和结尾的位置。\b匹配单词边界，如/\bJava\b/匹配任意位置的单词Java，但不匹配JavaScript。\B锚定与 非单词边界匹配的位置，如/\B[Ss]cript/匹配JavaScript、postscript但不匹配script或Scripting。

使用任意表达式作为锚定条件，(?=…)向前查找断言，其中的字符必须存在，但不会实际匹配。例如/[Jj]ava([Ss]cript)?(?=\:)/这个模式匹配Java或JavaScript后面跟冒号的情况（但是冒号不会在匹配结果中）。(?!…)否定式向前查找断言，要求必须不存在断言中指定的字符。如：/Java(?!Script)/表示匹配Java后面不是Script的字符。

向后查找断言，(?<=…)，向后查找否定断言(?<!…)

# 标志

符号 描述
g 全局匹配
i 表示模式匹配不区分大小写
m 多行模式

# 模式匹配的字符串方法

# search()

接收一个正则表达式作为参数，返回第一个匹配项起点字符的位置，没有找到返回-1。若参数不是正则表达式，会把这个参数传给RegExp构造函数转换为正则表达式。search()方法不支持全局搜索，其正则表达式参数中的g会被忽略。

"aaa".search(/a+/)//0
"aaa".search(/ab/)//-1

# replace()

接收一个正则表达式作为第一个参数，接收一个替换字符串作为第二个参数。如果正则表达式带全局标志g，则会替换调用字符串中的所有匹配项。如果replace()第一个参数不是正则表达式而是字符串，它不会调用RegExp构造函数转换为正则表达式，而是会按照字面值搜索。

replace()还可以引用正则表达式的括号分组。在替换字符串中用$加数字的形式进行引用。这两个字符就代表指定的子表达式，如下面这个例子，替换字符串参数 中通过$n引用了某个子表达式。替换过程还是会将匹配项与整个字符串进行替换。如果是命名捕获组，可以通过名字而非数字进行引用。

let reg = /(['"])([^'"]*)\1/
let str = 'He said "Hello man".'
str.replace(reg,'《$2》')//'He said 《Hello man》.'
str.replace(reg,'<$1$2$1>')//'He said <"Hello man">.'

let reg2 = /(?<quote>['"])(?<word>[^'"]*)\k<quote>/
str.replace(reg2,'**$<word>**')//'He said **Hello man**.'

第二个参数除了传替换字符串还可以传函数。该函数会接收到几个参数，第一个是匹配的整个文本，后面几个参数是捕获组匹配的子字符串，再接下来的参数是找到匹配项的位置，最后是调用replace方法的这个字符串。如果正则表达式有命名捕获组，还会收到一个参数，是一个对象，属性名是捕获组的名字，属性值是匹配的文本。

我认为这种场景一般用于将匹配项进行简单换算后替换的情况，即函数不宜过于复杂，最好采用简单的箭头函数。

let reg = /(['"])([^'"]*)\1/
let reg2 = /(?<quote>['"])(?<word>[^'"]*)\k<quote>/
let str = 'He said "Hello man".'
str.replace(reg,(a,b,c,d,e) => {
    console.log(`a ${a}\nb ${b}\nc ${c}\nd ${d}\ne ${e}\n`);
    return "喵喵喵"
})
/*
a "Hello man"
b "
c Hello man
d 8
e He said "Hello man".
'He said 喵喵喵.'
*/


//带命名捕获组的正则表达式
str.replace(reg2,(a,b,c,d,e,f) => {
    console.log(`a ${a}\nb ${b}\nc ${c}\nd ${d}\ne ${e}\n`,f);
    return "汪汪汪"
})
/*
a "Hello man"
b "
c Hello man
d 8
e He said "Hello man".
 {quote: '"', word: 'Hello man'}
'He said 汪汪汪.'
*/

# match()

接受一个正则表达式参数，如果参数不是正则表达式会调用RegExp构造函数转为正则表达式。匹配返回一个数组，如果没有匹配返回null。

支持全局搜索g，此时返回的数组中只包含所有匹配项。

没有全局搜索标志则只会查找第一个匹配项，返回的数组第一个元素是匹配的字符串，剩下的是括号分组的捕获组匹配的子字符串。非全局搜索的情况下返回的数组还有一些对象属性，input引用调用match的字符串；index是匹配项的起始位置；groups对应一个对象，属性名是命名捕获组的 名字，属性值是匹配的文本。

//解析URL的RegExp
let url = /(\w+):\/\/([\w.]+)\/(\S*)/;
let text = "Vist my blog at https://unleashj.github.io/123"
let match = text.match(url)
//['https://unleashj.github.io/123', 'https', 'unleashj.github.io', '123', index: 16, input: 'Vist my blog at https://unleashj.github.io/123', groups: undefined]

# matchAll()

接收一个带g标志的正则表达式，返回一个迭代器，每次迭代都产生一个与使用match得到的数组对象相同的对象。是循环遍历字符串中的所有匹配项的最简单最通用的方式。不会修改传入RegExp对象的lastIndex属性。

# split()

使用的传入的参数作为分隔符，将调用它的字符串拆分为子字符串保存在数组中。

# RegExp类

RegExp()可以接收一个或两个参数，第一个参数一般是一个字符串，包含正则表达式的主体表达式（即//之间的部分），第一个参数也可以是一个RegExp对象，用于复用之前的正则表达式。第二个参数是可选的，是标志位。RegExp构造函数主要用于动态创建正则表达式。

# test()

reg.test(str)如果字符串与模式匹配则返回true ，否则返回false。内部是调用了exec方法实现的。

# exec()

与字符串的match()方法类似。
