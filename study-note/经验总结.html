经验总结

2022/1/1

新开一个类别，专门记录一下平时遇到的一些问题

# v-if用在子组件身上，条件不成立时，props传入子组件的参数值受到影响。

# 问题：

子组件通过props接收一个数组对象dataSource，父组件data选项中设置dataSource初始值为空列表，在父组件的created钩子函数中为其赋值。子组件中有一个watch选项监听dataSource的变化，但没有使用immediate选项。调试过程中发现，watch的回调函数始终没有执行。

<div id="parent">
    <children v-if="dataSource.length>0" :dataSource="dataSource"></children>
</div>
<script>
    Vue.component("children", {
        props: ["dataSource"],
        template: "<h1>{{dataSource[0]}}</h1>",
        watch: {
            dataSource() {
                //cb
            },
        },
    });
    let vm = new Vue({
        el: "#parent",
        data: {
            dataSource: [],
        },
        created() {
            this.dataSource = [1, 2, 3, 4];
        }
    });
</script>

# 分析：

父组件初始化之后（created），dataSource已经赋值，此时还没有子组件实例。当子组件实例化时，在初始化阶段，子组件props上接收的参数dataSource已经有值（且不会再改变），因此watch的回调函数始终没有触发。基于此将父组件为dataSource赋值的逻辑放在了mounted钩子函数中。

做了上述修改之后，发现watch还是没有生效。看了下父组件模板中调用子组件的地方，在子组件标签中写了一个v-if=dataSource.length>0。到了父组件的模板编译阶段，解析到子组件标签时，因为父组件还没有挂载完成，dataSource还是空列表，v-if的条件不成立，所以并不会实例化子组件。当父组件挂载完成之后，给dataSource赋值，因为数据变化，会更新视图，v-if条件成立，此时再实例化子组件，dataSource依旧是有值，所以watch的回调函数仍然不会触发。

# 解决：

因为子组件是公共组件，不方便为watch添加immediate选项，解决方法是，在父组件data选项新增一个变量list，并在父组件created函数中，为list赋值（与dataSource赋一样的值），子组件标签中改为v-if=list.length>0。然后在父组件mounted函数中再为dataSource赋值。

这样修改之后，父组件初始化完毕（created），list已经有值，v-if条件成立，实例化子组件，此时子组件接收的dataSource prop是空值。接着子组件挂载完毕，父组件挂载完毕（mounted），为dataSource赋值，赋值之后因为数据更新，触发子组件中watch的回调函数。

<div id="parent">
    <children v-if="list.length>0" :dataSource="dataSource"></children>
</div>
<script>
    Vue.component("children", {
        props: ["dataSource"],
        template: "<h1>{{dataSource[0]}}</h1>",
        watch: {
            dataSource() {
                //cb
            },
        },
    });
    let vm = new Vue({
        el: "#parent",
        data: {
            dataSource: [],
            list: [],
        },
        created() {
            this.list = [1, 2, 3, 4];
        },
        mounted() {
            this.dataSource = this.list;
        },
    });
</script>

# 总结：
