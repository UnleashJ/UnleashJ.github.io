# 事件相关的实例方法

  与事件相关的方法有4个，分别是vm.$on vm.$once vm.$off vm.$emit，他们是在eventsMixin中挂载到Vue原型上，代码如下

export function eventsMixin (Vue) {
    Vue.prototype.$on = function (event,fn){}
    Vue.prototype.$once = function (event,fn){}
    Vue.prototype.$off = function (event,fn){}
    Vue.prototype.$emit = function (event){}

# vm.$on

  用法，详见Vue官网vm.$on( event, callback ) (opens new window)。监听当前实例上的自定义事件，事件可以由vm.$emit触发。回调函数会接收所有 传入事件所触发的函数的额度参数。

  在注册事件（vm.$on）时将回调函数收集起来，在触发事件（vm.$emit）时将收集起来的回调函数依次调用即可，代码如下。如果event是数组时，会遍历数组，将其中的每一项递归调用vm.$on。如果event不为数组，则向事件列表添加回调，就是将回调注册到事件列表中。

Vue.prototype.$on = function (event,fn){
    const vm = this
    if (Array.isArray(event)) {
        for (let i = 0, l = event.length; i < l; i++) {
            vm.$on(event[i], fn)
        }
    } else {
        (vm._events[event] || (vm._events[event] = [])).push(fn)
    }
    return vm
}

vm._events是一个对象，用来存储事件及其回调（事件列表）。vm._events是在_init（即initMixin）中创建的。

# vm.$off

  官方文档vm.$off (opens new window)。移除自定义事件监听器，有三种情况需要处理：

  • 如果没有提供参数，则移除所有的事件监听器；
  • 如果只提供了事件，则移除该事件所有的监听器；
  • 如果同时提供了事件与回调，则只移除这个回调的监听器。

  首先处理没有参数的情况，移除所有事件监听器，只需要将vm._events为初始状态就可以了。

Vue.prototype.$off = function (event,fn){
    const vm = this
    if(!arguments.length){
        vm._events = Object.create(null);
        return vm
    }
    return vm
}

  如果提供了事件，且事件是数组的话，需要遍历该事件数组，为每一个事件调用vm.$off即可。

Vue.prototype.$off = function (event,fn){
    const vm = this
    if(!arguments.length){
        vm._events = Object.create(null);
        return vm
    }
    
    if(Array.isArray(event)){
        for(let i=0,l=event.length;i<l;i++){
           this.$off(event[i],fn) 
        }
        return vm
    }
    return vm
}

  如果只提供了事件，则移除该事件所有的监听器，将vm._events中event属性对应的值重置为空即可。

Vue.prototype.$off = function (event,fn){
    const vm = this
    if(!arguments.length){
        vm._events = Object.create(null);
        return vm
    }
    
    if(Array.isArray(event)){
        for(let i=0,l=event.length;i<l;i++){
           this.$off(event[i],fn) 
        }
        return vm
    }
    
    let cbs = vm._events[event]
    if(!cbs){
        return vm
    }
    if(arguments.length === 1){
        vm._events[event] = null
        return vm
    }
    return vm
}

  如果同时提供了事件与回调，则只移除这个回调的监听器。遍历事件对应的事件列表（回调），在列表中找到与参数中提供的回调函数一直的那个函数，然后将它删除。

Vue.prototype.$off = function (event,fn){
    const vm = this
    if(!arguments.length){
        vm._events = Object.create(null);
        return vm
    }
    
    if(Array.isArray(event)){
        for(let i=0,l=event.length;i<l;i++){
           this.$off(event[i],fn) 
        }
        return vm
    }
    
    let cbs = vm._events[event]
    if(!cbs){
        return vm
    }
    if(arguments.length === 1){
        vm._events[event] = null
        return vm
    }
    
    if(fn){
        let consts = vm._events[event]
        let cb
        let i = cbs.length
        while(i--){
            cb = cbs[i]
            if(cb === fn || cb.fn === fn){
                cbs.splice(i,1)
                break
            }
        }
    }
    return vm
}

# vm.$once

  用法：vm.$once( event, callback ) (opens new window)。监听一个自定义事件，但是只触发一次。一旦触发之后，监听器就会被移除。当触发某一个事件之后，会调用监听事件时绑定的回调函数，我们可以在这个回调函数中将这个监听器移除，也就是调用vm.$off，实现如下：

Vue.prototype.$once = function (event,fn){
    const vm = this
    function on(){
        vm.$off(event,on)
        fn.apply(vm,arguments)
    }
    on.fn = fn//$off中是"if(cb === fn || cb.fn === fn)"，其中cb.fn === fn就是为这个准备的
    vm.$on(event,on)
    return vm
}

# vm.$emit

  用法：vm.$emit (opens new window)。触发当前实例上的事件。附加参数都会传给监听器回调。使用事件名从vm._events中取出对应的事件回调函数列表，然后依次调用各个回调函数。

Vue.prototype.$once = function (event,fn){
    const vm = this
    let cbs = vm._events[event]
    if(cbs){
        const args = toArray(arguments,1)
        for(let i=0,l=cbs.length;i<1;i++){
            try{
                cbs[i].apply(vm,args)
            }catch(e){
                handlerError(e,vm,`event handler for "${event}"`)
            }
        }
    }
    return vm
}

# 生命周期相关实例方法

  与生命周期相关的实例方法有4个，分别是vm.$mount vm.$forceUpdate vm.$nextTick vm.$destroy。

vm.$forceUpdate vm.$destroy是从lifecycleMixin中挂载到Vue原型上。vm.$nextTick是从renderMixin挂载到Vue原型上的。vm.$mount则是在跨平台的代码中挂载到原型上。

export function lifecycleMixin(Vue){
	 Vue.prototype.$forceUpdate = function () {}
	 Vue.prototype.$destroy = function () {}
}

export function renderMixin(Vue){
	 Vue.prototype.$nextTick = function () {}
}

# vm.$forceUpdate

  官方用法vm.$forceUpdate() (opens new window)。迫使 Vue 实例重新渲染，即手动通知Vue.js实例重新渲染。注意它仅仅影响实例本身和插入插槽内容的子组件，而不是所有子组件。每一个组件对应一个watcher，所以通过执行实例watcher的update方法，就可以使Vue.js重新渲染。实现方式如下：

Vue.prototype.$forceUpdate = function () {
    const vm = this
    if(vm._watcher){
        vm._watcher.update()
    }
}

# vm.$destroy

  用法vm.$destroy() (opens new window)。完全销毁一个实例。清理它与其它实例的连接，解绑它的全部指令及事件监听器。同时会触发 beforeDestroy 和 destroyed 的钩子。销毁组件的具体步骤如下：

  1. 首先在销毁之间触发beforeDestroy钩子函数
  2. 清理当前组件与父组件之间的链接。即将当前组件实例从父组件实例的$children属性中删除。

子组件在不同的父组件中是不同的Vue.js实例，所以一个子组件实例的父级只有一个。只需要销毁当前子组件实例即可。

  1. 销毁实例上的所有watcher，即将实例上所有的依赖追踪断掉，从状态的依赖列表中将watcher移除。

组件自身的watcher实例绑定在vm._watcher中。

  1. 销毁用户创建的watcher实例，即使用vm.$watch创建的watcher实例。

执行new Vue()时，在初始化的过程中会在this上添加一个_watchers属性。在Watcher类构造函数中，每当创建watcher实例时，都会将watcher实例添加到vm._wahtcers中。

export default class Watcher{
    constructor(vm,expOrFn,cb){
        vm._watchers.push(this)
        ...
    }
    ...
}

  1. 向实例中添加_isDestroyed属性表示Vue.js实例已经被销毁。

  2. 解绑模板中的所有指令。

  3. 触发destroyed钩子函数。

  4. 最后移除所有事件监听器。

调用vm.$off，只要不传任何参数，就可以移除所有事件监听器

完整代码如下：（其中有一些Vue后面新增的步骤代码）

Vue.prototype.$destroy = function () {
    const vm = this
    if (vm._isBeingDestroyed) {
        //表明vm实例正在被销毁，直接退出
        return
    }
    callHook(vm, 'beforeDestroy')//1.触发beforeDestroy钩子函数
    vm._isBeingDestroyed = true
    // 2.清理当前组件与父组件之间的链接。
    const parent = vm.$parent
    if (parent && !parent._isBeingDestroyed && !vm.$options.abstract) {
        //父组件存在且没有被销毁且不是抽象组件
        remove(parent.$children, vm)
    }
    // 3.销毁实例上的watcher
    if (vm._watcher) {
        vm._watcher.teardown()
    }
    // 4.销毁用户创建的watcher
    let i = vm._watchers.length
    while (i--) {
        vm._watchers[i].teardown()
    }
    // remove reference from data ob
    // frozen object may not have observer.
    if (vm._data.__ob__) {
        vm._data.__ob__.vmCount--
    }
    // 5.向实例中添加`_isDestroyed`属性表示Vue.js实例已经被销毁。
    vm._isDestroyed = true
    // 6.解绑模板中的所有指令。
    vm.__patch__(vm._vnode, null)
    // 7.触发destroyed钩子函数。
    callHook(vm, 'destroyed')
    // 8.最后移除所有事件监听器。
    vm.$off()
    // remove __vue__ reference
    if (vm.$el) {
        vm.$el.__vue__ = null
    }
    // release circular reference (#6759)
    if (vm.$vnode) {
        vm.$vnode.parent = null
    }
}

# vm.$nextTick

  官方文档vm.$nextTick (opens new window)接受一个回调函数作为参数，将回调延迟到下次 DOM 更新循环之后执行。它跟全局方法 Vue.nextTick 一样，不同的是回调的 this 自动绑定到调用它的实例上。如果没有提供回调且在支持 Promise 的环境中，则返回一个 Promise。vm.$nextTick其实就是将回调添加到微任务中。

  使用场景：更新了数据之后，需要对新DOM做一些操作。

  Vue中更新DOM的回调也是使用vm.$nextTick来注册到微任务中的，所以要想在vm.$nextTick中获取更新后的DOM，应该在更改数据之后使用vm.$nextTick注册回调。否则在回调中获取不到最新的DOM。

  Vue中vm.$nextTick和全局方法Vue.nextTick是相同的，所以nextTick抽象成了公共方法。vm.$nextTick会想微任务队列中添加回调，在一次事件循环中，如果反复调用vm.$nextTick，Vue.js不会将所有回调都依次添加到任务队列中，而是只添加一个任务，这一个任务触发时，会执行所有的回调函数。原理是这样的，Vue.js有一个列表来存储所有的vm.$nextTick参数中的回调函数，它是一个闭包，不会暴露给外部。当任务触发时，依次执行这个列表中的所有回调，并清空列表。代码如下：

import {nextTick} from '../util/index'
Vue.prototype.$nextTick = function (fn) {
    return nextTick(fn, this)
}

const callbacks = [];//存储回调函数的列表
let pending = false;//标志位，true代表已经向微任务队列中添加了任务，后续无需再次添加

//被注册（添加）到微任务队列中的那个任务。会遍历callbacks列表，执行所有回调函数
function flushCallbacks(){
    pending = false;//将标志位设置为false，即此时可以向微任务队列中再次注册任务
    const copies = callbacks.slice(0)//浅拷贝一份回调列表
    callbacks.length = 0;//将回调列表清空
    for(let i=0;i<copies.length;i++){
        copies[i]();
    }
}

//利用promise.then将flushCallbacks添加到微任务队列
let microTimerFunc
const p = Promise.resolve();
microTimerFunc = ()=>{
    p.then(flushCallbacks);
}

export function nextTick(cb,ctx){
    //向回调列表中添加方法
    callbacks.push(()=>{
        if(cb){
            cb.call(ctx)
        }
    })
    if(!pending){
        //如果标志位为false，此时可以向微任务队列中注册任务
        pending = true;//改变标志位，已经向微任务队列中添加了任务，后续无需再次添加
        microTimerFunc();//注册微任务       
    }
}

  并不是所有的环境都支持Promise，对于不支出的环境，Vue.js采用别的方式进行代替，如setImmediate、setTimeout、MutationObserver等，参考源码。

  官方文档中还有一句话，如果没有提供回调且在支持 Promise 的环境中，则返回一个 Promise。也就是说可以这样使用vm.$nextTick

this.$nextTick().then(function (){
	...
})

  实现方式如下

export function nextTick (cb, ctx) {
    let _resolve//是个闭包，在调用的时候它已经有值了
    callbacks.push(() => {
        if (cb) {
            cb.call(ctx)
        } else if (_resolve) {
            _resolve(ctx)
        }
    })
    if (!pending) {
        pending = true
        microTimerFunc()
    }
    if (!cb && typeof Promise !== 'undefined') {
        return new Promise(resolve => {
            _resolve = resolve
        })
    }
}

  完整源码如下（翻译一下注释）：

/* @flow */
/* globals MutationObserver */

import { noop } from 'shared/util'
import { handleError } from './error'
import { isIE, isIOS, isNative } from './env'

export let isUsingMicroTask = false

const callbacks = []
let pending = false

function flushCallbacks () {
  pending = false
  const copies = callbacks.slice(0)
  callbacks.length = 0
  for (let i = 0; i < copies.length; i++) {
    copies[i]()
  }
}

// Here we have async deferring wrappers using microtasks.
// In 2.5 we used (macro) tasks (in combination with microtasks).
// However, it has subtle problems when state is changed right before repaint
// (e.g. #6813, out-in transitions).
// Also, using (macro) tasks in event handler would cause some weird behaviors
// that cannot be circumvented (e.g. #7109, #7153, #7546, #7834, #8109).
// So we now use microtasks everywhere, again.
// A major drawback of this tradeoff is that there are some scenarios
// where microtasks have too high a priority and fire in between supposedly
// sequential events (e.g. #4521, #6690, which have workarounds)
// or even between bubbling of the same event (#6566).
let timerFunc

// The nextTick behavior leverages the microtask queue, which can be accessed
// via either native Promise.then or MutationObserver.
// MutationObserver has wider support, however it is seriously bugged in
// UIWebView in iOS >= 9.3.3 when triggered in touch event handlers. It
// completely stops working after triggering a few times... so, if native
// Promise is available, we will use it:
/* istanbul ignore next, $flow-disable-line */
if (typeof Promise !== 'undefined' && isNative(Promise)) {
  const p = Promise.resolve()
  timerFunc = () => {
    p.then(flushCallbacks)
    // In problematic UIWebViews, Promise.then doesn't completely break, but
    // it can get stuck in a weird state where callbacks are pushed into the
    // microtask queue but the queue isn't being flushed, until the browser
    // needs to do some other work, e.g. handle a timer. Therefore we can
    // "force" the microtask queue to be flushed by adding an empty timer.
    if (isIOS) setTimeout(noop)
  }
  isUsingMicroTask = true
} else if (!isIE && typeof MutationObserver !== 'undefined' && (
  isNative(MutationObserver) ||
  // PhantomJS and iOS 7.x
  MutationObserver.toString() === '[object MutationObserverConstructor]'
)) {
  // Use MutationObserver where native Promise is not available,
  // e.g. PhantomJS, iOS7, Android 4.4
  // (#6466 MutationObserver is unreliable in IE11)
  let counter = 1
  const observer = new MutationObserver(flushCallbacks)
  const textNode = document.createTextNode(String(counter))
  observer.observe(textNode, {
    characterData: true
  })
  timerFunc = () => {
    counter = (counter + 1) % 2
    textNode.data = String(counter)
  }
  isUsingMicroTask = true
} else if (typeof setImmediate !== 'undefined' && isNative(setImmediate)) {
  // Fallback to setImmediate.
  // Technically it leverages the (macro) task queue,
  // but it is still a better choice than setTimeout.
  timerFunc = () => {
    setImmediate(flushCallbacks)
  }
} else {
  // Fallback to setTimeout.
  timerFunc = () => {
    setTimeout(flushCallbacks, 0)
  }
}

export function nextTick (cb?: Function, ctx?: Object) {
  let _resolve
  callbacks.push(() => {
    if (cb) {
      try {
        cb.call(ctx)
      } catch (e) {
        handleError(e, ctx, 'nextTick')
      }
    } else if (_resolve) {
      _resolve(ctx)
    }
  })
  if (!pending) {
    pending = true
    timerFunc()
  }
  // $flow-disable-line
  if (!cb && typeof Promise !== 'undefined') {
    return new Promise(resolve => {
      _resolve = resolve
    })
  }
}

# vm.$mount (opens new window)

  官方链接vm.$mount (opens new window)（好好看看用法及示例）。如果 Vue 实例在实例化时没有收到 el 选项，则它处于“未挂载”状态，没有关联的 DOM 元素。可以使用 vm.$mount() 手动地挂载一个未挂载的实例。

  Vue.js的完整版与只包含运行时版本之间的差异在于是否有编译器。在只包含运行时版本的vm.$mount 没有编译步骤，默认实例上已存在渲染函数，如果不存在则会设置一个。在完整版中，vm.$mount 会首先检查template或el选项所提供的模板是否已经转换成渲染函数，如果没有则进入编译过程，将模板编译成渲染函数，完成之后再进入挂载与渲染的流程中。

# 1. 完整版vm.$mount的实现原理

  在下面的代码中，先将Vue原型上的$mount方法保存在mount中，以便后续使用。然后Vue原型上的$mount方法被一个新方法覆盖。新方法中调用mount，这种做法叫作函数劫持。mount中是$mount方法的核心功能，完整版中需要新增编译功能。

  el参数支持元素类型（Node/Element）或者字符串类型的选择器，首先通过el参数获取DOM元素。接着实现编译器的功能。先判断实例中是否有渲染函数，只有不存在时才需要将模板编译成渲染函数。详情见代码中的注释。

若存在渲染函数render又给出了template，此时因为不需要编译模板，所以template是无效的

const mount = Vue.prototype.$mount;
Vue.prototype.$mount = function(el){
    el = el && query(el)//1

    const options  = this.$options
    if(!options.render){
        //不存在渲染函数，需要将模板编译成渲染函数并赋值给options.render
        let template = options.template
        if(template){
            if(typeof template ==="string"){
                if(template.charAt(0) === '#'){
                    //如果template是以#开头，则认为他是id选择符
                    template = idToTemplate(template)
                }
                //如果不是以#开头的字符串，则认为他是用户自己定义的模板（HTML字符串），不用做解析
            }else if(template.nodeType){
                //如果template有nodeType属性，则他是DOM元素，使用innerHTML作为模板
                template = template.innerHTML
            }else{
                //警告模板非法
                warn('模板非法' + template);
                return this
            }
        }else{
            //用户没提供模板，使用el获取HTML字符串作为模板
            template = getOuterHTML(el)
        }

        // 模板编译逻辑
        if(template){
            const {render} = compileToFunctions(
                template,
                {},//一个对象，里面是有内容的，此处不做研究
                this
            )
            options.render = render
        }

    }
    return mount.call(this,el)
}

function query(el){
    //返回参数el对应的DOM元素
    if(typeof el === 'string'){
        // 如果el是一个字符串，可以通过选择器方法获取对应DOM元素
        const selected = document.querySelector(el)
        if(!selected){
            // 如果不存在对应DOM元素，则新建一个空的div元素并返回
            return document.createElement('div')
        }
    }else{
        // 如果el不是字符串，则默认它是一个元素类型的节点
        return el
    }
}

function getOuterHTML(el){
    if(el.outerHTML){
        return el.outerHTML
    }else{
        let container = document.createElement('div')
        container.appendChild(el.cloneNode(true))
        return container.innerHTML
    }
}

function idToTemplate(id){
    const el = query(id)
    return el && el.innerHTML
}

function compileToFunctions(template,options,vm){
    options = extend({},options)//混合到空对象中

    // 检查缓存避免无用功
    const key = options.delimiters
    ? String(options.delimiters) + template
    : template;
    if (cache[key]) {
        return cache[key]
    }

    // 编译
    const compiled = compile(template, options);//代码字符串保存在compiled.render中
    const res = {};
    res.render = createFunction(compiled.render);
    return (cache[key] = res)
}

function createFunction(code){
    return new Function(code)
}

function mountComponent(vm,el){
    if(!vm.$options.render){
        // 如果没有render渲染函数，使用createEmptyVNode作为渲染函数，执行后生成一个注释类型的VNode节点
        vm.$options.render = createEmptyVNode
    }
    callHook(vm,'beforeMount');//触发生命周期钩子

    /*  
        vm._update(vm._render()):
        先试用渲染函数得到一份最新的VNode节点树
        然后通过_update方法对新旧VNode做对比并更新DOM节点
    */
    vm._watcher = new Watcher(vm,()=>{
        vm._update(vm._render())
    },noop)//noop  no option

    callHook(vm,'mounted')
    return vm
}

  watcher将持续观察模板中用到的所有数据（状态），当这些数据变化时，watcher将收到通知，从而进行渲染（即执行vm._update(vm._render())）操作。

  因为此时vm._update(vm._render())就是watcher中的getter函数，第一次执行时，触发数据的getter，收集watcher进依赖列表。后续数据变化时，会再次调用watcher的getter函数，也就是执行vm._update(vm._render())，即进行渲染（Watcher原理详见《Vue源码-变化侦测（响应式原理）》）。

# Vue.extend (opens new window)

  使用基础 Vue 构造器，创建一个“子类”。参数是一个包含组件选项的对象。data 选项是特例，需要注意 - 在 Vue.extend() 中它必须是函数。其实就是创建了一个Sub函数并继承了父级，如果直接使用 Vue.extend()，则Sub继承于Vue构造函数。Vue中源码如下：

Vue.cid = 0
let cid = 1

/**
   * Class inheritance
   */
Vue.extend = function (extendOptions: Object): Function {
    extendOptions = extendOptions || {}
    const Super = this
    const SuperId = Super.cid
    //增加缓存策略，反复调用Vue.extend返回的是同一个结果。
    const cachedCtors = extendOptions._Ctor || (extendOptions._Ctor = {})
    if (cachedCtors[SuperId]) {
        return cachedCtors[SuperId]
    }
	
    //检查name是否符合规范
    const name = extendOptions.name || Super.options.name
    if (process.env.NODE_ENV !== 'production' && name) {
        validateComponentName(name)
    }
	
    //Sub就是要返回的子类（构造函数）
    const Sub = function VueComponent (options) {
        this._init(options)
    }
    //Sub的原型对象是Super的一个实例
    Sub.prototype = Object.create(Super.prototype)
    Sub.prototype.constructor = Sub
    Sub.cid = cid++
    Sub.options = mergeOptions(
        Super.options,
        extendOptions
    )
    Sub['super'] = Super

    // For props and computed properties, we define the proxy getters on
    // the Vue instances at extension time, on the extended prototype. This
    // avoids Object.defineProperty calls for each instance created.
    if (Sub.options.props) {
        initProps(Sub)
    }
    if (Sub.options.computed) {
        initComputed(Sub)
    }

    // allow further extension/mixin/plugin usage
    Sub.extend = Super.extend
    Sub.mixin = Super.mixin
    Sub.use = Super.use

    // create asset registers, so extended classes
    // can have their private assets too.
    ASSET_TYPES.forEach(function (type) {
        Sub[type] = Super[type]
    })
    // enable recursive self-lookup
    if (name) {
        Sub.options.components[name] = Sub
    }

    // keep a reference to the super options at extension time.
    // later at instantiation we can check if Super's options have
    // been updated.
    Sub.superOptions = Super.options
    Sub.extendOptions = extendOptions
    Sub.sealedOptions = extend({}, Sub.options)

    // cache constructor
    cachedCtors[SuperId] = Sub
    return Sub
}

function initProps (Comp) {
    const props = Comp.options.props
    for (const key in props) {
        proxy(Comp.prototype, `_props`, key)
    }
}

function initComputed (Comp) {
    const computed = Comp.options.computed
    for (const key in computed) {
        defineComputed(Comp.prototype, key, computed[key])
    }
}

# Vue.directive (opens new window)

Vue.directive只是注册或获取全局指令，而不是让指令生效。所以我们只需要把指令保存在某个位置，后续需要让指令生效时再从这个位置取出来执行。

  注册指令代码实现如下：（参考官方用法进行理解）

Vue.options = Object.create(null);
Vue.options['directives'] = Object.create(null);

Vue.directive = function(id,definition){
    if(!definition){
        //不存在definition说明是获取全局指令
        return this.options['directives'][id]
    }else{
        //注册
        if(typeof definition === 'function'){
            //如果definition是个函数，则默认监听bind和update两个方法
            definition = {bind:definition,update:definition}
        }
        //definition不是函数，则是用户自定义的指令对象，直接保存即可
        this.options['directives'][id] = definition
        return definition;
    }
}

# Vue.component (opens new window)

  注册或获取全局组件，与Vue.directive作用类似，仅仅是注册或获取全局组件。definition支持两种参数类型，函数或者对象。组件其实是一个构造函数，是使用Vue.extend生成的子类。所以要使用Vue.extend将definition统一处理成构造函数，方法如下：

Vue.options['components'] = Object.create(null);

Vue.directive = function(id,definition){
    if(!definition){
        //不存在definition说明是获取全局组件
        return this.options['components'][id]
    }else{
        //注册
        if(isPlainObject(definition)){
            //如果definition是一个对象，则将其转为构造函数
            definition.name = definition.name || id
            definition = Vue.extend(definition) 
        }
        this.options['components'][id] = definition
        return definition;
    }
}

# Vue.use (opens new window)

  安装 Vue.js 插件。如果插件是一个对象，必须提供 install 方法。如果插件是一个函数，它会被作为 install 方法。install 方法调用时，会将 Vue 作为参数传入。该方法需要在调用 new Vue() 之前被调用。当 install 方法被同一个插件多次调用，插件将只会被安装一次。

import { toArray } from '../util/index'


Vue.use = function (plugin) {
    //判断插件是否已经注册过
    const installedPlugins = (this._installedPlugins || (this._installedPlugins = []))
    if (installedPlugins.indexOf(plugin) > -1) {
        return this
    }

    // 除第一参数以外，剩余的所有参数赋给args
    const args = toArray(arguments, 1)
    //将Vue添加到最前面
    args.unshift(this)
    if (typeof plugin.install === 'function') {
        //如果plugin是对象且有install方法
        plugin.install.apply(plugin, args)
    } else if (typeof plugin === 'function') {
        //如果plugin是函数
        plugin.apply(null, args)
    }
    installedPlugins.push(plugin)
    return this
}

export function toArray (list, start){
    start = start || 0
    let i = list.length - start
    const ret: Array<any> = new Array(i)
    while (i--) {
        ret[i] = list[i + start]
    }
    return ret
}
