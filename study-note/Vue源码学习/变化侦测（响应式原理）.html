# 变化侦测（响应式原理）

  利用了设计模式中的观察者模式。观察者（Observer）模式的定义：指多个对象间存在一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并被自动更新。

  Vue.js 2.0中，引入了虚拟DOM，通常一个状态绑定的依赖是一个组件，这样状态更新时触发的依赖数量变少，不会因为更新粒度过细而导致性能消耗严重。所以当状态发生改变时，会将通知发送到组件，由组件内部在通过虚拟DOM重新渲染。（相对较细的更新粒度是一个状态绑定着很多个依赖，每一个依赖都是一个DOM节点。）

  Vue是在实例初始化阶段对data选项中的数据（data对象）转为响应式的。

# Object的变化侦测

  JavaScript中侦测一个对象的变化可以使用Object.defineProperty和Proxy，因为彼时Proxy在浏览器中的支持并不理想，所以Vue2使用前者实现变化侦测。在Vue3中已经改为Proxy实现响应式系统。

  我的要做的是当某个组件使用了某个状态时，将这个组件作为状态的一个依赖收集起来。当这个状态发生改变时（被重新赋值），通知这个状态对应的所有依赖进行更新。

  由此我们可以使用Object.defineProperty()，将状态的属性重新定义为访问器属性，即使用getter和setter来侦听他的变化。在getter中，将用到某数据的依赖收集起来，并返回它的值；在setter中设置新的值并通知这些依赖进行更新。

  可以使用简单的数组保存状态的依赖，但这样写有点耦合，我们可以定义一个Dep类，使用它专门管理数据的依赖。我们将依赖在全局的window.target中也保存一份（此时先不用关心依赖是什么）。所以Dep的实现代码如下：

export class Dep {
    constructor() {
        this.subs = [];
    }

    addSup(sub) {
        //添加依赖
        this.subs.push(sub);
    }

    removeSub(sub) {
        //删除依赖
        remove(this.subs, sub);
    }

    depend() {
        // 收集依赖
        if (window.target) {
            this.addSup(window.target);
        }
    }

    // 更新依赖，遍历所有依赖依次更新，sub就是window.target，update是它的一个方法
    notify() {
        let subs = this.subs.slice();
        for (let i = 0, l = subs.length; i < l; i++) {
            subs[i].update();
        }
    }
}

//删除数组中的某一项
function remove(arr, item) {
    if (arr.length) {
        let index = arr.indexOf(item);
        if (index !== -1) {
            return arr.splice(index, 1);
        }
    }
}

  如前所述，我们可以在getter中，将用到某数据的依赖收集起来，并返回它的值；在setter中设置新的值并通知这些依赖进行更新。定义响应式状态的代码如下：

function defineReactive(data, key, val) {
    // 定义依赖的容器
    let dep = new Dep();
    //将data对象的key属性定义为访问器属性，值为val
    Object.defineProperty(data, key, {
        enumerable: true,//可枚举
        configurable: true,//可使用delete删除
        get: function () {
            dep.depend();//收集依赖
            return val;
        },
        set: function (newVal) {
            if (val === newVal) {
                return;
            }
            //当依赖绑定的数据更新时候，触发依赖更新 
            dep.notify();
            val = newVal;
        },
    });
}

  依赖通常是一个组件（即模板中使用了状态）。但我们经常使用computed和watch（计算属性和侦听器）来侦听某个状态的变化，此时也会创建依赖，因此我们可以抽象一个Watcher类（每一个依赖对应一个watcher实例）。数据发生变化时通知watcher，再由他执行update方法通知其他地方。以下面的vm.$watch为例，监听data.a.b.c的变化，发生改变时候，执行第二个参数中的回调函数：

vm.$watch('a.b.c', function (newVal, oldVal) {
  // 做点什么
})

  Watcher类的实现代码如下

class Watcher{
    constructor(vm,expOrFn,cb){
        this.vm = vm;
        // 执行this.getter()就是读取data中某数据的值，以此触发该数据的getter
        this.getter = parsePath(expOrFn);
        this.cb = cb
        this.value = this.get()//取值，触发getter，收集依赖
    }

    get(){
        window.target = this
        // 触发数据的getter，以此收集依赖
        let value = this.getter.call(this.vm,this.vm)
        // 收集依赖后将window.targert置为undefined
        window.target = undefined
        return value;

    }

    update(){
        let oldValue = this.value
        this.value = this.get()//得到新值
        this.cb.call(this.vm,this.value,oldValue)
    }
}

//匹配不是简单的键路径
let bailRE = /[^\w.$]/
function parsePath(path){
    if(bailRE.test(path)){
        return
    }
    //传进来的path是个字符串，不是属性名
    const segement = path.split('.')
    return function(obj){
        for (let i = 0; i < segement.length; i++) {
            if(!obj) return
            obj = obj[segement[i]]
        }
        return obj
    }
}

  定义一个Observer类型，将数据内的所有属性（包括子属性）都转换成访问器属性，追踪他们的变化（添加getter/setter）。

  • Observer
export class Observer {
    constructor(value) {
        this.value = value;
        if (!Array.isArray(value)) {
            // 不是数据绑定
            this.walk(value);
        }
    }

    walk(obj) {
        /*  
       	 	forin枚举可枚举属性，包含继承的可枚举属性
        	Object.keys()返回对象可枚举自有属性名的数组
    	*/
        let keys = Object.keys(obj);
        for (let i = 0; i < keys.length; i++) {
            let key = keys[i];
            defineReactive(obj, key, obj[key]);
        }
    }
}

function defineReactive(data, key, val) {
    // 如果数据的某个属性值也是一个对象，则将其也转为访问器属性。递归
    if (typeof val === "object") {
        new Observer(val);
    }
    let dep = new Dep();
    Object.defineProperty(data, key, {
        enumerable: true,
        configurable: true,
        get: function () {
            dep.depend();
            return val;
        },
        set: function (newVal) {
            if (val === newVal) {
                return;
            }
            dep.notify();
            val = newVal;
        },
    });
}

  经过上面的讨论，只有经过getter收集依赖，setter触发依赖更新，vue才可以侦测某个数据的变化。所以，若在方法中直接给某个数据添加属性，因为没有新属性的getter和setter，所以新属性值的变化无法侦测到。

new Vue({
    el: "#app",
    mounted() {
        this.obj.name = "unleash";
    },
    data: {
        obj: {}
    },
});

  若是直接把某一个对象赋值为一个新对象，因会data.obj属性已经是访问器属性，此时会触发obj属性的setter，所以仍然可以实现变化侦测。这也就是为什么使用this.someObject = Object.assign({}, this.someObject, { a: 1, b: 2 })可以实现响应式原理的原因，因为其本质也是赋值一个新对象给someObject。

 new Vue({
     el: "#app",
     mounted() {
         this.obj = {
             name: "unleash",
         };
     },
     data: {
         obj: {},
     },
 });

  以上只是实现了对象的响应式系统，相对较为简单。但Vue 源码中所作的处理要更多些，其中一些是关于数组的处理，数组需要经过其他特殊的处理才能实现变化侦测。

# Array的变化侦测

# 创建拦截器

  Vue2中，通过push、pop等Array原型上的方法来改变数组内容时，并不会触发setter。因此Vue使用了一个拦截器覆盖Array.prototype，在拦截器中添加可以响应数组变化的方法，然后再使用原生Array.prototype的方法去操作数组。

  可以改变数组的原型方法有这7个：push\pop\shift\unshift\splice\sort\reverse。

// 设置数组原型Array.prototype的拦截器arrayMethods

// 保存一下Array原型上的所有方法（函数劫持）
const arrayProto = Array.prototype

// 以Array.prototype为原型对象创建拦截器对象
export const arrayMethods = Object.create(arrayProto)

const methodsToPatch = [
    'push',
    'pop',
    'shift',
    'unshift',
    'splice',
    'sort',
    'reverse'
]

methodsToPatch.forEach(function (method){
    // 保存原始方法
    let original = arrayProto[method];
    //向拦截器中添加相关方法
    Object.defineProperty(arrayMethods,method,{
        value:function mutator(...args){
            // 这里发送变化通知
            return original.apply(this,...args)
        },
        // 不可枚举，可修改可删除
        enumerable:false,
        writable:true,
        configurable:true
    })
})

  不能直接用拦截器去覆盖所有数组的Array.prototype，避免污染全局Array，应该只拦截那些被侦测（需求侦测）变化的数组，也就是只覆盖那些响应式数组的原型。修改Observer，添加代码如下：

export default class Observer {
    constructor(value) {
        this.value = value;
        if (Array.isArray(value)) {
            // 如果响应式数据是数组，将改数组对象的原型设置为arrayMethods（拦截器）
            Object.setPrototypeOf(value,arrayMethods);
        }else{
            this.walk(value);
        }
    }
    ...
}

因为Object.setPrototypeOf()以及__proto__各种浏览器中的支持情况不一定，所以Vue源码中对这种情况作了特殊处理。这样Observer的构造函数中，首先会判断value的类型，若是一个数组，则拦截其原型上的7个方法，否则是一个其他对象，调用walk，将其转为响应式数据。

export default class Observer {
    constructor(value) {
        this.value = value;
        if (Array.isArray(value)) {
            //判断是否支持__proto__
            if (hasProto) {
                protoAugment(value, arrayMethods)
            } else {
                //不支持的话就将arrayMethods的所有方法复制到被侦测的数组上
                copyAugment(value, arrayMethods, arrayKeys)
            }
        } else {
            this.walk(value)
        }
    }
    ...
}
    
function protoAugment (target, src: Object) {
  /* eslint-disable no-proto */
  target.__proto__ = src
  /* eslint-enable no-proto */
}


function copyAugment (target: Object, src: Object, keys: Array<string>) {
  for (let i = 0, l = keys.length; i < l; i++) {
    const key = keys[i]
    def(target, key, src[key])
  }
}

# 收集依赖

  现在假设data选项中有一个list属性，它是一个数组。在初始化阶段的initState时，会为data生成一个Observer实例。因为data是一个对象，所以调用walk方法，将data中的每一个属性转为访问器属性（即将data转为响应式数据）。因此我们就可以将数组的依赖在list属性的getter中收集起来。

data:{
	list:[1,2,3,4]
}

  当数组变化时，我们要触发依赖更新，也就是在拦截器方法中通知变化。若只像普通Object对象那样，将依赖收集在defineReactive里定义的那个dep中，后续在拦截器中无法得到这个依赖列表（如下）。

function defineReactive(data, key, val) {
    if (typeof val === "object") {
        new Observer(val);
    }
    let dep = new Dep();
    Object.defineProperty(data, key, {
        enumerable: true,
        configurable: true,
        get: function () {
            dep.depend();
            return val;
        },
        set: function (newVal) {
            if (val === newVal) {
                return;
            }
            dep.notify();
            val = newVal;
        },
    });
}

  拦截器中可以直接获得的是当前调用方法的数组，也就是value，因此在Vue中，增加了如下操作。我们在value中定义一个属性__ob__指向当前Observer实例，并且给这个Observer实例定义一个dep属性。这样在拦截器中就可以通过array.__ob__.dep获得数组的依赖列表，并触发更新。这样对于数组，应该在其Observer实例中的依赖列表中也收集一次依赖。修改代码如下

import Dep from './dep.js'
import arrayMethods from './arrayMethods.js'
import { def } from '../vue/src/core/util/lang.js';

export default class Observer {
    constructor(value) {
        this.value = value;
        //将当前observer实例添加到value上
        def(value,'__ob__',this)
        //在Observer实例中新建一个依赖列表
        this.dep = new Dep();
        if (Array.isArray(value)) {
            // 如果响应式数据是数组，将改数组对象的原型设置为arrayMethods（拦截器）
            Object.setPrototypeOf(value,arrayMethods);
        }else{
            this.walk(value);
        }
    }

    walk(obj) {
        /*  
        forin枚举可枚举属性，包含继承的可枚举属性
        Object.keys()返回对象可枚举自有属性名的数组
        */
        let keys = Object.keys(obj);
        for (let i = 0; i < keys.length; i++) {
            let key = keys[i];
            defineReactive(obj, key, obj[key]);
        }
    }
}


function defineReactive(data, key, val) {
    // 如果数据的某个属性的值也是一个对象，给这个子对象也定义一个Observer实例
    let childOb = observe(val)
    // 定义依赖的容器
    let dep = new Dep();
    Object.defineProperty(data, key, {
        enumerable: true,
        configurable: true,
        // 给数据绑定getter
        get: function () {
        /* 
        收集触发getter函数时，与数据绑定的依赖
        （假设依赖是一个函数，保存在window.target） 
      	*/
            dep.depend();
            // 如果有observer实例，向子属性的observer实例的dep中也添加依赖
            if(childOb){
                childOb.dep.depend()
            }
            return val;
        },
        set: function (newVal) {
            if (val === newVal) {
                return;
            }
            /*  
        当依赖绑定的数据更新时候，触发依赖更新
      */
            dep.notify();
            val = newVal;
        },
    });
}

function observe(value,asRootDara){
    if(!isObject(value)){
        return 
    }
    let ob;
    if(value.hasOwnProperty('__ob__') && value.__ob__ instanceof Observer){
        ob = value.__ob__
    }else{
        ob = new Observer(value)
    }
    return ob
}

function isObject(value){
    return value!== null && typeof value === 'object'
}

function def(obj,key,val,enumerable){
    Object.defineProperty(obj, key, {
        value: val,
        enumerable: !!enumerable,
        writable: true,
        configurable: true
    })
}

  这样在拦截器中，我们就可以通过this（数组）拿到observer实例，进而拿到依赖列表，然后出发依赖更新。

methodsToPatch.forEach(function (method){
    // 保存原始方法
    let original = arrayProto[method];
    def(arrayMethods,method,function mutator(...args){
        let result = original.apply(this,...args)
        let ob = this.__ob__
        ob.dep.notify()// 这里发送变化通知
        return original.apply(this,...args)
    })
})

经过上面的操作，虽然是为了实现数组的变化侦测，但是对于普通的子对象其实也会在value.__ob__.dep中收集一份依赖。这样做虽然牺牲了空间复杂度，但是能够较好的实现各种数据的变化侦测。

# 侦测数组中元素的变化

  数组身上的某个属性值发生变化，也需要发生通知，使用下面的方法递归侦听数组中子数据的变化。这样就会为其子数据（当子数据是一个object时）新增一个Observer实例。

export default class Observer {
    constructor(value) {
        this.value = value;
        def(value,'__ob__',this)
        this.dep = new Dep();
        if (Array.isArray(value)) {
            // 如果响应式数据是数组，将改数组对象的原型设置为arrayMethods（拦截器）
            Object.setPrototypeOf(value,arrayMethods);
            //侦测数组中每一项的变化
            this.observeArray(value)
        }else{
            this.walk(value);
        }
    }
    ...
    observeArray(items) {
        for (let i = 0, l = items.length; i < l; i++) {
            observe(items[i]);
        }
    }
}

  经过上面的处理，虽然会为数组元素中的每个对象数据新增一个Observer实例和依赖列表，但是这个依赖列表并没有收集依赖的逻辑，因为我们不能像属性getter那样拦截数组元素的访问。所以还需要再增加下面的逻辑

function defineReactive(data, key, val) {
    let childOb = observe(val)
    let dep = new Dep();
    Object.defineProperty(data, key, {
        enumerable: true,
        configurable: true,
        get: function () {
            dep.depend();
            if(childOb){
                childOb.dep.depend()
                //如果是一个数组，遍历数组中的元素，通知其依赖列表收集依赖
                if (Array.isArray(value)) {
                    dependArray(value);
                }
            }
            return val;
        },
        set: function (newVal) {
            if (val === newVal) {
                return;
            }
            dep.notify();
            val = newVal;
        },
    });
}

function dependArray (value) {
    for (var e = (void 0), i = 0, l = value.length; i < l; i++) {
        e = value[i];
        e && e.__ob__ && e.__ob__.dep.depend();
        if (Array.isArray(e)) {
            dependArray(e);
        }
    }
}

# 侦测新增元素

  如果method是push\unshift\splice 这种可以新增数组元素的方法，那么从args中将新增元素取出来，暂存在inserted中。

methodsToPatch.forEach(function (method){
    // 保存原始方法
    let original = arrayProto[method];
    def(arrayMethods,method,function mutator(...args){
        let result = original.apply(this,args)
        let ob = this.__ob__
        // 侦测新增数据的变化
        let inserted
        switch(method){
            case 'push':
            case 'unshift':
                inserted = args
                break
            case 'splice':
                inserted = args.slice(2);
                break
        }

        if(inserted){
            ob.observeArray(inserted)
        }

        ob.dep.notify()// 这里发送变化通知
        return result
    })
})

# 相关API实现原理

# vm.$watch (opens new window)

# 1. 用法及注意事项

  请先了解API的用法，可以查看Vue的官方文档。这里提示几个注意点：

  1. expOrFn若是表达式，则只接受以点分隔的路径，例如a.b.c，如果是一个复杂的表达式可以用函数代替表达式。
  2. vm.$watch返回一个可以取消观察的函数，用来停止触发回调。
  3. 选项deep是为了发现对象内部值的变化，监听数组的变动不需要这么做。

# 2. 内部原理

  该方法定义在Vue的原型对象上，内部其实是对Watcher的一种封装：

export Vue.prototype.$watch = function(expOrFn,cb,options){
    let vm = this;
    options = options || {};
    let watcher = new Watcher(vm,expOrFn,cb,options)
    if(options.immediate){
        cb.call(vm,watch.value)
    }
    // 返回取消监听的函数
    return function unwatchFn(){
        watcher.teardown()
    }
}

  expOrFn的类型是表达式或者函数，在Watcher类中需要修改一下对于expOrFn的处理逻辑。如果expOrFn是函数，直接将其赋值给this.getter。接着在vm实例对象上调用this.getter时，会触发所有相关数据的getter，因此也都会被Watcher观察，即相关依赖（watcher对象）会添加到依赖列表dep中。这些数据中任何一个发生变化时，watcher都会得到通知。

export default class Watcher{
    constructor(vm,expOrFn,cb){
        this.vm = vm;

# 4. deep参数实现原理

  deep参数实现原理：除了要触发当前这个被监听数据的收集依赖的逻辑之外，还要把当前监听的这个值在内的所有子值都触发一遍收集依赖逻辑（其实就是取一下值，触发相关数据的getter，进而收集当前依赖）。

export default class Watcher{
    constructor(vm,expOrFn,cb,options){
        this.vm = vm;
        this.deps = [];//保存dep
        this.depIds = new Set();//保存dep的id

        if(options){
            this.deep = !!options.deep
        }else{
            this.deep = false
        }

        // 执行this.getter()就可以读取data中某数据的值，以此触发该数据的getter
        if(typeof expOrFn === 'function'){
            this.getter = expOrFn
        }else{
            this.getter = parsePath(expOrFn);
        }  
        this.cb = cb
        this.value = this.get()//拿到值
    }

    get(){
        window.target = this
        // 触发数据的getter，以此收集依赖
        let value = this.getter.call(this.vm,this.vm)
        if(this.deep){
            // 递归value，以触发子值的getter
            traverse(value)
        }
        // 收集依赖后将window.targert置为undefined
        window.target = undefined
        return value;

    }

let seenObjects = new Set();//保存已收集依赖的depId
export function traverse(val){
  _traverse(val,seenObjects)
  seenObjects.clear();
}

function _traverse(val,seen){
  let i, keys
  const isA = Array.isArray(val)
  if ((!isA && !isObject(val)) || Object.isFrozen(val) || val instanceof VNode) {
    return
  }
  // 如果数据身上已经有依赖列表，则直接用
  if (val.__ob__) {
    const depId = val.__ob__.dep.id
    // 不会重复触发依赖
    if (seen.has(depId)) {
      return
    }
    seen.add(depId)
  }
  if (isA) {
    i = val.length
    // 数组这样读数据不会触发getter
    while (i--) _traverse(val[i], seen)
  } else {
    keys = Object.keys(val)
    i = keys.length
    // 这里读数据会触发getter
    while (i--) _traverse(val[keys[i]], seen)
  }
}

  下面解释下监听数组为什么不需要deep。我们知道收集依赖的逻辑其实就是触发一下对应属性的getter，如果是数组，由于dependArray逻辑的存在，数组元素的依赖列表也会收集这个watcer，所以不需要deep参数也可以深度监听。如果是对象，只有dep和childOb.dep两个依赖列表收集了这个watcher，所以无法触发子对象的getter。

function defineReactive(data, key, val) {
    let childOb = observe(val)
    let dep = new Dep();
    Object.defineProperty(data, key, {
        enumerable: true,
        configurable: true,
        get: function () {
            dep.depend();
            if(childOb){
                childOb.dep.depend()
                //如果是一个数组，遍历数组中的元素，通知其依赖列表收集依赖
                if (Array.isArray(value)) {
                    dependArray(value);
                }
            }
            return val;
        },
        set: function (newVal) {
            if (val === newVal) {
                return;
            }
            dep.notify();
            val = newVal;
        },
    });
}

# vm.$set

  官网介绍 vm.$set (opens new window)。注意：target不能是Vue.js实例或者Vue.js实例的根数据对象。

  当target是数组时，使用数组原型拦截器上的splice方法即可实现。具体相关实现可以参看下面的代码。

export default $set = function(target,key,val){
    //target是数组
    if (Array.isArray(target) && isValidArrayIndex(key)) {
        //key小于length，说明是要给key这个下标的元素这只一个新值，删除原值，再新增
        //key大于length，说明是新增一个值
        target.length = Math.max(target.length, key)
        target.splice(key, 1, val)
        return val
    }

    //对象已经存在key，直接设置
    if (key in target && !(key in Object.prototype)) {
        target[key] = val
        return val
    }
    const ob = target.__ob__
    //注意对象不能是 Vue 实例，或者 Vue 实例的根数据对象。
    if (target._isVue || (ob && ob.vmCount)) {
        process.env.NODE_ENV !== 'production' && warn(
        'Avoid adding reactive properties to a Vue instance or its root $data ' +
        'at runtime - declare it upfront in the data option.'
        )
        return val
    }
    //如果对象不是响应式数据，直接设置值
    if (!ob) {
        target[key] = val
        return val
    }
    // 最后给这个值添加变化侦测
    defineReactive(ob.value, key, val)
    ob.dep.notify()
    return val 
}

# vm.$delete

vm.$delete (opens new window)，target不能是Vue.js实例或者Vue.js实例的根数据对象。原理就是删除属性后向依赖发消息。

export function $delete(target,key,val){
    if (Array.isArray(target) && isValidArrayIndex(key)) {
        target.splice(key, 1)
        return
    }
    const ob = target.__ob__
    if (target._isVue || (ob && ob.vmCount)) {
        process.env.NODE_ENV !== 'production' && warn(
            'Avoid deleting properties on a Vue instance or its root $data ' +
            '- just set it to null.'
        )
        return
    }
    if (!hasOwn(target, key)) {
        return
    }
    delete target[key]
    //非响应式数据，直接删除就可以
    if (!ob) {
        return
    }
    ob.dep.notify()
}

let bailRE = /[^\w.$]/
//传进来的path是个字符串，不是属性名
function parsePath(path){
    if(bailRE.test(path)){
        return
    }
    const segement = path.split('.')
    return function(obj){
        for (let i = 0; i < segement.length; i++) {
            if(!obj) return
            obj = obj[segement[i]]
        }
        return obj
    }
}
