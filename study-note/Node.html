Nodejs

2022/1/2 Node

Node.js是一个基于V8引擎的JavaScript运行环境。在已经了解JavaScript语法的基础上，学习Nodejs中的内置模块。

# fs模块

# 读取文件

利用fs模块读取文件fs.readFile()，接收三个参数：文件路径、编码（可选）、会带上文件数据（以及错误）进行调用的回调函数。使用fs.readFile()方法是异步读取文件内容，也可以使用fs.readFileSync同步读取文件内容。

const fs = require("fs");

console.log(1)

fs.readFile("./text.txt", "utf-8", (err, data) => {
    if (err) {
        console.error(err);
        return;
    }
    console.log(data);
});

console.log(2)

try {
    const data = fs.readFileSync("/Users/joe/test.txt", "utf8");
    console.log(data);
} catch (err) {
    console.error(err);
}

console.log(3)

以上代码的输出结果为：

1
2
123456
3
123456

# 写入文件

利用fs.writeFile()异步写入文件，接收三个参数：文件路径、写入内容、编码格式（可选）、回调函数（参数只有一个err）。同步的版本 fs.writeFileSync()。

  1. 只能创建文件不能创建目录。
  2. 这两个 API 会替换文件的内容（如果文件已经存在）
// 异步写入文件
fs.writeFile('./textWrite.txt','自定义内容','utf-8',err=>{
    if(err){
        console.log(err)
    }
})

//同步写入文件
try {
    fs.writeFileSync("./testWriteSync.txt", "自定义内容");
} catch (err) {
    console.error(err);
}

# 相对路径的问题

在上面读取或写入文件时，使用的文件路径是相对路径，这依赖于终端正确的路径（执行node命令时所处的目录），即相对路径是相对于终端的路径进行动态拼接的，虽然使用绝对路径可以解决这个问题，但是这样代码不利于书写和维护。

__dirname表示当前文件所处的目录，因此操作文件时，可以使用以下方式拼接路径。注意此时相对路径的.或..也没了。

fs.readFile(__dirname + "/text.txt", "utf-8", (err, data) => {
    if (err) {
        console.error(err);
        return;
    }
    console.log(data);
});

# path模块

# path.join()拼接路径

path.join()，连接路径的两个或多个部分，将多个路径片段拼接为完整的路径字符串，且支持相对路径的写法，即../会取消前面的路径。

const path = require('path')
let pathStr = path.join('/users', './notes.txt')//\users\notes.txt

因此fs中探讨的路径问题应该使用path.join()拼接：

const path = require('path')
const fs = require("fs");

fs.readFile(path.join(__dirname,'./text.txt'), "utf-8", (err, data) => {
    if (err) {
        console.error(err);
        return;
    }
    console.log(data);
});

# path.basename()获取文件名

path.basename()返回路径的最后一部分，第二个参数（可选）可以过滤掉文件的扩展名（注意要带.）

let bathname = path.basename(path.join(__dirname,'../fs/text.txt'),'.txt')
console.log('bathname: ',bathname)//bathname: text

# path.extname()获取文件扩展名

返回路径的扩展名部分。

let extname = path.extname(path.join(__dirname,'../fs/text.txt'))
console.log('extname:',extname)//extname: .txt

# Node中的模块

Node中的模块分为以下三类：

  1. 内置模块：fs\path\http等
  2. 自定义模块：用户创建的js文件
  3. 第三方模块：需要先下载

require()加载模块时，会执行被加载模块中的代码。使用require加载模块时，可以省略后缀名。

模块作用域

module对象，每个模块都有一个module对象，其中的exports属性指向的对象可以向外共享成员变量，供外界使用，require导入的也就是这个对象。

Module {
    id: '.',
    path: 'D:\\study\\node\\path',
    exports: {},
    parent: null,
    filename: 'D:\\study\\node\\path\\exercise.js',
    loaded: false,
    children: [],
    paths: [
        'D:\\study\\node\\path\\node_modules',
        'D:\\study\\node\\node_modules',
        'D:\\study\\node_modules',
        'D:\\node_modules'
    ]
}

默认情况下，exports与module.exports指向同一个对象。最终共享的结果，还是以module.exports指向的对象为准。因此若我们改变了exports或module.exports其中一个指向的对象，他们两个指向的就不再是同一个对象，导出时会以module.exports为准。
